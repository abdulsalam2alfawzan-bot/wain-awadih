# سيناريوهات QA

## رحلة العميل

1. بدء مسار مشكلة بوصف واضح لا يحتاج أسئلة.
2. وصف قصير يحتاج سؤالين أو ثلاثة.
3. وصف غير كافٍ يعرض عدم كفاية المعلومات دون اختلاق مشكلة.
4. فقدان فرامل يحول التوجيه إلى خطر وخدمة السطحة.
5. رائحة وقود ودخان كثيف يمنعان طمأنة الاستمرار في القيادة.
6. اختيار شركة ثم التحقق من تغير قائمة الموديلات.
7. اختيار شركة أخرى وموديل آخر وظهور الحقول اليدوية.
8. إخفاء الممشى في القطع والسطحة.
9. جعل سنة الصنع اختيارية للسطحة.
10. طلب الموقع الدقيق إلزاميًا للسطحة واختياريًا للبقية.
11. مسار قطع الغيار دون حقل اسم القطعة.
12. مسار صيانة دورية مع خدمة وملاحظة.
13. إنشاء الطلب ثم الإحالة قبل بطاقة الشريك.
14. عدم وجود تطابق في مدينة غير مغطاة.
15. إعادة تحميل الصفحة أثناء كل مرحلة واستعادة المسودة.
16. نسخ رابط الطلب والتأكد من عدم وجود الجوال فيه.
17. فتح واتساب برسالة تحتوي الاسم الأول ورقم الطلب والسيارة والاحتياج.
18. طلب شريك ثانٍ وثالث مع استبعاد السابق.
19. منع الشريك الرابع.
20. تأكيد الوصول دون احتساب رسم.
21. تأكيد بدء الاستقبال بعد الوصول واحتساب رسم واحد.
22. منع تكرار الرسم بعد إعادة النقر.
23. منع التقييم قبل تجربة مؤكدة.
24. إرسال تقييم موثق ومنع تكراره.
25. إغلاق طلب غير نشط إداريًا بعد خمسة أيام.

## بوابة الشريك

26. الدخول بحساب WA-DEMO / 1234.
27. تحديث حالة الإحالة إلى وصل العميل.
28. تحديثها إلى بدأ الاستقبال أو الفحص.
29. تقديم اعتراض على الإحالة وتعليق الرسم.
30. تقديم اعتراض على تقييم ووضعه تحت المراجعة.
31. إنشاء خصم مع نسبة وخدمات وشروط ومدة.
32. إيقاف الخصم.
33. إصدار مطالبة عند بلوغ 100 ريال.
34. إصدار مطالبة شهرية دون 100 ريال لسجل من شهر سابق.
35. تحويل مطالبة متأخرة إلى overdue وتعليق الشريك.
36. التأكد من استبعاد الشريك الموقوف من المطابقة.
37. تسجيل دفع تجريبي وإصدار إيصال.
38. إعادة تفعيل الشريك بعد تسوية كل المطالبات المفتوحة.

## الانضمام

39. تسجيل ورشة مع تخصصات وشركات.
40. تسجيل مركز صيانة وخدمات متعددة.
41. تسجيل محل قطع دون مخزون.
42. تسجيل سطحة مع مناطق تغطية وأنواع مركبات.
43. إضافة خصم اختياري والتحقق من التواريخ.
44. منع طلب مكرر بنفس المنشأة والجوال.
45. متابعة طلب الانضمام بالرابط الخاص.

## الجودة التقنية

46. فحص جميع ملفات JavaScript عبر `node --check`.
47. فحص الروابط المحلية وعدم وجود ملفات مفقودة.
48. فحص معرفات HTML المكررة.
49. فحص تسمية جميع الحقول.
50. فحص RTL وViewport وCSP في كل صفحة.
51. فحص توازن أقواس CSS ودعم reduced-motion.
52. تقديم جميع الصفحات والملفات الأساسية عبر HTTP بحالة 200.
