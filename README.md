# وين أوديها — دليل التشغيل

مشروع ويب عربي متجاوب مبني باستخدام HTML وCSS وJavaScript فقط. تعمل الصفحات مباشرة على استضافة ثابتة، وتستخدم `localStorage` لحفظ البيانات والعلاقات بين الطلبات والشركاء والإحالات والتقييمات والرسوم والفواتير والمدفوعات.

## التشغيل محليًا

يفضل تشغيل المشروع من خادم محلي بدل فتح الملفات مباشرة:

```bash
python -m http.server 8080
```

ثم افتح:

```text
http://localhost:8080/index.html
```

يمكن كذلك استخدام إضافة Live Server في Visual Studio Code.

## الصفحات الأساسية

- `index.html`: الصفحة الرئيسية والمسارات.
- `customer.html`: إنشاء طلب العميل للمشكلة أو قطع الغيار أو السطحة أو الصيانة.
- `track.html`: متابعة الطلب والتواصل وطلب شريك آخر والتقييم.
- `join.html`: طلب انضمام الشريك بنموذج متكيف مع نوع النشاط.
- `join-status.html`: متابعة طلب الانضمام.
- `workshop-login.html`: دخول الشريك.
- `workshop-dashboard.html`: إدارة الإحالات والحالات والتقييمات والخصومات والاعتراضات والفواتير والمدفوعات والملف.
- `payment.html`: تسجيل سداد الفاتورة محليًا.
- `receipt.html`: عرض إيصال السداد.
- `privacy.html`: سياسة الخصوصية.
- `terms.html`: الشروط وحدود المسؤولية.

## التعديلات المطبقة في هذه النسخة

- إظهار التواصل مع الشريك مباشرة بعد الخصم.
- وضع رابط الطلب في نهاية صفحة النتيجة والمتابعة، مع النسخ وطلب شريك آخر.
- إضافة زري الرئيسية والسابق أعلى جميع الصفحات، وإرشاد قصير أسفل كل صفحة.
- تخصيص ساعات العمل لكل يوم، بفترة أولى وفترة ثانية اختيارية.
- خصومات تشمل جميع الخدمات أو خدمات مختارة، مع تاريخ انتهاء إلزامي عند عدم الاستمرار.
- نموذج سطحة مخصص لبطاقة التشغيل واللوحة والتغطية والمركبات الممكن نقلها.
- حقول مستقلة لمراكز الصيانة ومحلات قطع الغيار.
- ربط أقسام لوحة الشريك كلها ببيانات `localStorage`.
- تحسين العرض المتجاوب والحالات الفارغة والنجاح والخطأ.

## تشغيل الاختبارات

من داخل مجلد المشروع:

```bash
node tests/qa-unit.js
python tests/static_audit.py
python tests/targeted_patch_audit.py
```

لفحص بناء JavaScript:

```bash
for f in assets/*.js; do node --check "$f"; done
```

يوجد اختبار متصفح في:

```text
tests/browser_e2e.py
```

ويتطلب Python وPlaywright ومتصفح Chromium في بيئة تسمح بفتح العناوين المحلية.

## النشر

المشروع جاهز للنشر على استضافة ثابتة مثل GitHub Pages أو Netlify. توجد الملفات:

- `.nojekyll`
- `netlify.toml`
- `_headers`

## ملاحظات تشغيلية

- الحفظ محلي داخل المتصفح والجهاز نفسه.
- مسح بيانات الموقع من المتصفح يحذف البيانات المحلية.
- لا تتم مزامنة البيانات بين الأجهزة دون خادم خلفي.
- زر WhatsApp يفتح رسالة جاهزة، ولا يرسلها تلقائيًا.
- صفحة السداد تسجل العملية داخل المشروع ولا تنفذ خصمًا مصرفيًا.
