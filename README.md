# منصة «وين أوديها» — النموذج التفاعلي المتكامل

مشروع عربي RTL متعدد الصفحات، مبني بـ **HTML وCSS وJavaScript فقط**، ويعمل دون خادم خلفي أو إطار عمل. صُمم لعرض نموذج تشغيل «وين أوديها» واختبار رحلة العميل والورشة والانضمام والرسوم محليًا قبل ربطه بخدمات الإنتاج.

## التشغيل السريع

1. فك ضغط المشروع كاملًا.
2. افتح `index.html` مباشرة في المتصفح.
3. للحصول على أفضل تجربة، شغّل المجلد عبر خادم ثابت أو ارفعه إلى GitHub Pages أو Netlify.

مثال اختياري عبر Python:

```bash
python -m http.server 8080
```

ثم افتح `http://localhost:8080`.

## بيانات العرض التجريبية

### متابعة طلب عميل

- رقم الطلب: `WA-10425`
- رقم الجوال: `0500000001`

### دخول الورشة

- الجوال: `0550000101`
- رمز الدخول: `1234`

### الدفع التجريبي

- رقم بطاقة تجريبي: `4242 4242 4242 4242`
- اسم حامل البطاقة: أي اسم من 3 أحرف فأكثر
- تاريخ الانتهاء: `12/30`
- رمز التحقق: `123`

لا تتم أي عملية مالية حقيقية، ولا يُحفظ رقم البطاقة أو رمز التحقق؛ يُحفظ آخر أربعة أرقام فقط داخل الإيصال التجريبي.

## الصفحات

- `index.html`: الصفحة الرئيسية التسويقية.
- `customer.html`: رحلة العميل لمساري الصيانة الدورية والمشكلة.
- `track.html`: استرجاع الطلب، واتساب، تأكيد النتيجة، طلب ورشة بديلة، سجل الإحالات، والتقييم.
- `workshop-login.html`: دخول الورشة التجريبي.
- `workshop-dashboard.html`: الإحالات والتقييمات ومؤشرات الأداء وكشف الحساب.
- `payment.html`: محاكاة سداد رسوم المنصة.
- `receipt.html`: إيصال الدفع التجريبي والطباعة.
- `join.html`: طلب انضمام ورشة متعدد الخطوات.
- `join-status.html`: متابعة حالة طلب الانضمام.
- `404.html`: صفحة خطأ مناسبة للاستضافة الثابتة.

## التخزين المحلي

يستخدم المشروع `localStorage` لحفظ:

- طلبات العملاء وبيانات السيارات.
- الإحالات وحالاتها وأسباب طلب البديل.
- تقييمات الورش الموثقة.
- الرسوم والمدفوعات والإيصالات التجريبية.
- طلبات انضمام الورش والمسودات.
- جلسة دخول الورشة التجريبية.

يمكن حذف بيانات الموقع من إعدادات المتصفح لإعادة البداية، أو استخدام زر «إعادة البيانات التجريبية» في لوحة الورشة.

## قواعد التشغيل المطبقة

- لا توجد أسعار إصلاح متوقعة أو نطاقات تكلفة.
- لا يقدم النظام تشخيصًا نهائيًا.
- تظهر ورشة واحدة فقط في كل مرة.
- تُنشأ الإحالة وتُحفظ قبل إظهار بيانات الورشة.
- يمكن طلب ورشة بديلة داخل نفس الطلب، بحد أقصى 3 إحالات.
- تُستبعد الورش السابقة من المطابقة داخل الطلب نفسه.
- تتم المطابقة التجريبية حسب المسار والمدينة.
- التقييم مرتبط برقم طلب وإحالة مؤكدة.
- رسوم المنصة ثابتة وتجريبية، ولا ترتبط بقيمة إصلاح السيارة.

## النشر عبر HTTPS

### GitHub Pages

1. ارفع جميع الملفات إلى جذر المستودع.
2. افتح **Settings → Pages**.
3. اختر النشر من فرع `main` والمجلد `/root`.
4. سيصدر GitHub رابط HTTPS تلقائيًا.

يتضمن المشروع ملف `.nojekyll` لتقليل مشكلات النشر الثابت.

### Netlify

اسحب المجلد إلى Netlify أو اربطه بالمستودع. ملف `netlify.toml` وملف `_headers` مرفقان بإعدادات أمنية أساسية.

## حدود النموذج الحالي

هذا نموذج Front-end تجريبي، لذلك:

- التحليل والمطابقة محاكاة محلية وليسا ذكاءً اصطناعيًا متصلًا بخدمة خارجية.
- بيانات الورش وأرقام واتساب تجريبية ويجب استبدالها قبل الإطلاق.
- لا يوجد تسجيل دخول حقيقي أو OTP.
- لا توجد قاعدة بيانات مركزية؛ البيانات خاصة بالمتصفح والجهاز الحالي.
- لا يوجد دفع حقيقي أو بوابة دفع.
- رفع الوثائق يحفظ أسماء الملفات فقط ولا يرفعها إلى خادم.

## ما يلزم قبل الإطلاق الفعلي

ربط قاعدة بيانات وخادم خلفي، تفعيل OTP، ربط خدمة ذكاء اصطناعي بضوابط واضحة، اعتماد سياسة الخصوصية والموافقات، إدخال بيانات ورش حقيقية موثقة، ربط واتساب بالأرقام الصحيحة، وتطبيق بوابة دفع معتمدة عند اعتماد النموذج التجاري.

راجع `AUDIT_REPORT.md` و`TEST_REPORT.md` للتفاصيل، و`SOURCE_CODE.md` لنسخة مجمعة من الكود الكامل.
