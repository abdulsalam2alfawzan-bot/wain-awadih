# وين أوديها — MVP تجريبي متكامل

منصة ويب عربية Mobile-First وRTL لتوجيه صاحب السيارة إلى شريك واحد مناسب، مع رحلة كاملة للعميل وبوابة للشريك ومحاكاة منطق البيانات والرسوم عبر `localStorage`.

> جميع أسماء الشركاء وأرقامهم ومواقعهم وتقييماتهم في بيانات Seed **تجريبية وليست لشركاء حقيقيين**.

## نطاق النسخة

تشمل النسخة أربعة مسارات:

1. **عندي مشكلة في السيارة**: وصف عامي، تحليل محلي قائم على قواعد، صفر إلى ثلاثة أسئلة غالبًا، توقع احتمالي، تخصص، استعجال، وخطوة تالية.
2. **محل قطع غيار**: الدلالة على محل مناسب لنوع السيارة دون طلب اسم القطعة أو تأكيد المخزون.
3. **سطحة**: المدينة، الموقع الدقيق، حالة حركة السيارة، والوجهة عند معرفتها.
4. **صيانة دورية**: اختيار خدمة الصيانة والممشى والمدينة.

## التشغيل محليًا

يمكن فتح `index.html` مباشرة، لكن يوصى بتشغيل خادم محلي حتى تتطابق التجربة مع الاستضافة:

```bash
python -m http.server 8080
```

ثم افتح:

```text
http://localhost:8080/
```

بديل Node.js:

```bash
npx serve .
```

## نموذج البيانات

تستخدم النسخة المفاتيح التالية فقط بوصفها طبقة بيانات مترابطة:

```text

wa_partners

wa_ratings
wa_objections
wa_discounts
wa_fees
wa_claims
wa_payments
wa_join_applications
wa_consents
wa_sessions
wa_meta
```

تتضمن طبقة التخزين:

- إصدار بيانات وترحيل آمن من النسخة السابقة.
- Seed غير هدّام.
- معرفات مرجعية بين الجداول.
- منع تكرار العملاء والسيارات والرسوم والتقييمات.
- فحص سلامة العلاقات.
- تواريخ إنشاء وتحديث وإغلاق.
- تعقيم النصوص قبل التخزين والعرض.

## قواعد العمل الرئيسية

- لا يظهر أكثر من شريك واحد في الوقت نفسه.
- ينشأ الطلب وتسجل الإحالة قبل عرض الشريك.
- يستبعد كل شريك سبق ترشيحه للطلب.
- الحد الأقصى ثلاثة شركاء للطلب.
- لا تعرض أسعار تقديرية.
- لا يقدم محرك AI تشخيصًا نهائيًا أو توصية باستبدال قطعة.
- لا تختلق المطابقة مدينة أو مسافة أو توفرًا.
- يستحق الرسم فقط بعد **الوصول + بدء الاستقبال أو الفحص**.
- التقييم مرتبط بطلب وإحالة وتجربة مؤكدة.
- يغلق الطلب إداريًا بعد خمسة أيام من آخر نشاط دون حذف السجلات.
- تصدر المطالبة عند 100 ريال أو التسوية الشهرية.
- يتوقف الشريك المتأخر عن السداد عن الظهور في المطابقة حتى التسوية.

## تشغيل الاختبارات

اختبارات منطق البيانات والأعمال:

```bash
node tests/qa-unit.js
```

التدقيق الثابت للصفحات والروابط والوصولية الأساسية:

```bash
python tests/static_audit.py
```

فحص تركيب جميع ملفات JavaScript:

```bash
for file in assets/*.js; do node --check "$file"; done
```

## إعادة ضبط البيانات التجريبية

من أدوات المطور في المتصفح:

```javascript
localStorage.clear();
location.reload();
```

سيعيد النظام Seed التجريبي دون الحاجة إلى تنزيل جديد.

## الاستضافة

### GitHub Pages

1. ارفع محتويات المجلد إلى جذر المستودع.
2. افتح **Settings → Pages**.
3. اختر النشر من الفرع الرئيسي والمجلد `/root`.
4. تأكد من عمل الموقع عبر HTTPS.

ملف `.nojekyll` موجود لتجنب معالجة Jekyll.

### Netlify

ارفع المجلد مباشرة أو اربطه بمستودع GitHub. ملف `netlify.toml` ورؤوس الأمان جاهزة.

## حدود النسخة التقنية

هذه نسخة عرض وتشغيل Frontend فقط. قبل الإطلاق التجاري يجب استبدال المحاكاة بما يلي:

- Backend وقاعدة بيانات مركزية.
- OTP حقيقي وربط رسائل الجوال.
- مصادقة آمنة للشركاء.
- تكامل WhatsApp Business API عند الحاجة للرسائل الآلية.
- سجل تدقيق مركزي وصلاحيات إدارية.
- تشفير ونظام نسخ احتياطي ومراقبة.
- مراجعة قانونية واعتماد سياسات الخصوصية والشراكة.

`localStorage` خاص بالمتصفح والجهاز، ولا يزامن البيانات بين مستخدمين أو أجهزة مختلفة.
